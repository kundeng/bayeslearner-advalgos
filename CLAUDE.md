# Project: Learn Advanced Algorithms

A Quarto book of companion notes for **Harvard COMPSCI 224 — Advanced Algorithms**, generated by converting YouTube lecture transcripts into self-contained chapters.

## Working agreements

- The book lives at `bayeslearner-advalgos/`. Each chapter is one lecture, named `lectures/NN-<slug>.qmd`.
- The notes are **not transcripts**. They reorganise lectures freely for clarity, prefer intuition before math, and target a grad audience with CLRS-level algorithms, probability, and mathematical maturity.
- Build target: HTML + PDF via Quarto. TikZ for math/algorithm diagrams, Mermaid for simple flowcharts without math, Manim for the 1–2 hardest concepts per lecture (algorithm animations, convergence, geometric intuition).
- **Visualization is critical.** Algorithms are spatial and sequential — aim for one diagram every 400–600 words. Prefer step-by-step state diagrams over abstract architecture boxes.
- For now, treat all `python` code blocks as **non-executable** (plain ` ```python `, not ` ```{python} `). When we wire up a working conda env we'll switch executable chunks back on.

## The conversion prompt — load this before any chapter work

When the user asks you to **draft, refine, or critique a chapter** for this book, the source of truth for *how* a chapter should be written is `prompts/lecture-to-qmd.qmd`. **Read that file first.** It defines:

- the chapter's required structure (front matter → why-it-matters → body → cheat sheet → FAQ → further reading → exercises),
- voice, math, diagram, callout, and code conventions,
- length targets and anti-patterns,
- placeholder variables to fill in for this course.

The prompt is **course-agnostic** with `{{PLACEHOLDER}}` variables. For this project, use:

| Placeholder | Value |
|-------------|-------|
| `{{COURSE_ID}}` | COMPSCI 224 |
| `{{COURSE_TITLE}}` | Advanced Algorithms |
| `{{TERM}}` | *(check transcript year)* |
| `{{AUDIENCE_DESCRIPTION}}` | A grad student with undergraduate algorithms (CLRS-level), probability, linear algebra, and mathematical maturity. Comfortable with asymptotic notation, amortised analysis, and basic randomised algorithms. |
| `{{AUTHOR}}` | Kun Deng |
| `{{CODE_STACK_DESCRIPTION}}` | Python with `numpy` and pseudocode as the default stack. Use actual implementations only when they illuminate the algorithm's behaviour — pseudocode is preferred for clarity. |

Treat the prompt as binding. If the user requests something that conflicts with it, flag the conflict and propose either updating the prompt or making a one-off exception — don't silently diverge.

## Workflow for a new lecture

1. **Get the transcript.** Two paths:
   - **User pastes it** (the reliable default).
   - **Auto-download** via `download_transcripts.py <playlist_or_video_url> transcripts/`. YouTube rate-limits aggressively — use a Tailscale exit node or different network if blocked (HTTP 429). **Try one video first** before the whole playlist.
   - If the transcript API fails, try `yt-dlp --write-auto-sub --sub-lang en --skip-download`.
2. Save the raw transcript under `transcripts/`.
3. Open `prompts/lecture-to-qmd.qmd`, fill in the placeholders, follow it strictly.
4. Output the chapter to `lectures/NN-<slug>.qmd`.
5. Add the chapter to `_quarto.yml` under the appropriate part.
6. Build with `quarto render`.
7. Read the rendered chapter end-to-end before declaring it done.

## TikZ setup (already solved in the NLP sibling project)

- **Engine:** Each `.qmd` file with TikZ diagrams needs `engine: knitr` in its YAML front matter.
- **Dependencies:** R + knitr + rmarkdown + magick + pdftools + TinyTeX. Install with:
  ```bash
  sudo apt install r-base-core
  Rscript -e 'install.packages(c("knitr","rmarkdown","magick","pdftools"), lib=Sys.getenv("R_LIBS_USER"), repos="https://cloud.r-project.org")'
  quarto install tinytex
  ```
- **TikZ block syntax** — use knitr chunk headers, NOT `%%|` comments:
  ```
  ```{tikz, label="fig-name", fig.cap="Caption text", echo=FALSE, out.width="90%"}
  \usetikzlibrary{arrows.meta, positioning}
  \begin{tikzpicture}[...]
  ...
  \end{tikzpicture}
  ```
  ```
- **Gotchas:**
  - Do NOT put `\usepackage` inside tikz blocks — knitr handles the preamble.
  - Only `\usetikzlibrary` is allowed inside the block.
  - `\mathbb` does NOT work — use `\mathbf{R}` for real number sets.
  - `echo: false` in YAML front matter hides TikZ source from readers.
  - If a chapter has both `{tikz}` and `{python}` blocks, knitr handles Python via the `reticulate` R package.

## Known build / tooling issues

- **Dropbox file-locks** on `.quarto/_freeze` and `site_libs/` produce "process cannot access the file" errors *after* a successful render. The HTML output is fine; ignore the trailing error or copy `_book/` to `/tmp/` for preview.
- **YouTube rate-limiting:** Both `youtube-transcript-api` and `yt-dlp` get HTTP 429 from this machine's home IP. Use a Tailscale exit node (`sudo tailscale set --exit-node=<node>`) to route through a different IP. Remember to disable it after (`sudo tailscale set --exit-node=`).

## Style reminders

- **Every equation that gets referenced later** needs `{#eq-name}`, referred to as `@eq-name`.
- **Every figure** needs both a `label` and a `fig-cap`.
- The cheat sheet is a *summary*, not a *duplication* of the body.
- The FAQ pulls real questions from the transcript's Q&A — never invent.
- Add a `callout-note` titled "Aside (mine, not the lecturer's)" whenever extending beyond what the lecture said.
- **For algorithm diagrams:** show the data structure state before and after each step, annotate with intermediate values ($d[v]$, flow, potential, etc.).
